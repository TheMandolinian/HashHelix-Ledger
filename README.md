## Core Concepts (v1.7 terminology)

To avoid confusion between the engine and the artifacts, HashHelix uses the following terms going forward:

- **Singularity**  
  The deterministic core engine of the HashHelix Ledger.  
  It implements the œÄ/n-phase-drifted recurrence and produces a fully reproducible temporal state sequence.

- **Time Shards** (internal)  
  Internal units of temporal progression generated by the Singularity.  
  Time Shards are *not* user-facing assets; they only exist inside the engine as part of the hashing and state evolution process.

- **Temporal Relics** (formerly ‚Äúshards‚Äù)  
  User-facing artifacts created by the Singularity.  
  A Temporal Relic is a signed, versioned object that can represent a dataset, experiment, benchmark run, or narrative artifact.  
  Relics do not contain the engine; they reference it.

- **Relic Lineages**  
  Each Temporal Relic can act as the root of a lineage (a branch of meaning or application logic).  
  Lineages define how Relics relate to each other over time (for example, a Genesis Relic spawning derived Relics).

- **Genesis Temporal Relic 000000**  
  The first Temporal Relic created in the HashHelix ecosystem.  
  It is a legendary artifact and a symbolic origin for Relic lineages, but it does not replace the Singularity or act as a second engine.
